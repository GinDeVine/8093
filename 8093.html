ABOUT

Hiding data within the opacity of images, starting at pixel (x,y) with an interval of z(i) between every pixel. Unused pixel get a slight static noise of opacity. The opacity is barely - if at all - noticable by the bare eye, and even though you know it's there, you'll need both x, y and z to find it.

As I assume you are using Google Chrome, pop open your JavaScript console and type in the code mentioned below. Firefox users can do it with FireBug and people who doesn not realize their own good and use any other browser might need to type it into the URL bar with "javascript: " in front.

To hide a message, use the function hideData(DATA [, x, y, z]). The data is, surprisingly, the data you want to hide, and the optional xyz is the coordinate to begin and interval, as mentioned above. X and y is numbers and z is a function of the pattern "function(i){return i*2}" (this example hides the data in every other pixel). If x, y or z is undefined, the default values will be used instead.

In order to read your hidden message, use the function readFromPNG(SOURCE [, x, y, z]). The source being the source of the image and the xyz the same as was used to hide the data. If not source specified, it will use the image displayed, and if no image is displayed, it'll throw a nice little error.
